# 🧠 LEO 知识库构建思路

> 本文档记录这个 Obsidian 知识库的构建背景、设计决策、目录分层原则以及使用约定。
> 如果你是一个 AI 智能体，请先阅读本文，再对库中的内容进行操作。

---

## 1. 缘起

这个知识库的最初起点是一个**交易日志**。用户希望通过 Obsidian 系统化地记录每次交易，然后通过周总结、月总结进行复盘，并长期沉淀交易规则。

在整理过程中，发现原有的笔记结构比较杂乱（学习笔记、比赛记录、生活代办、图片散落混在一起），于是决定**推倒重来**，建立一个可以承载「生活·学习·事业·投资」一体化的个人知识库。

---

## 2. 核心设计原则

```
原则 1：先记录，再整理 —— 降低起步门槛
原则 2：沉淀可复用的信息，不保留临时状态
原则 3：模板尽量简洁，保证长期可坚持
原则 4：目录不超过三层，超过则创建新笔记引用
```

---

## 3. 顶层架构：九大领域

整个知识库根目录下只有 9 个带编号的顶层目录，以及一个 `首页.md` 入口文件。

```text
00_Inbox           # 收件箱 — 临时存放，待处理
01_个人成长         # 阅读、技能、自我提升
02_专业事业         # 工作、项目、履历、面试
03_财富投资         # 资产总览、交易日志、复盘规则
04_工具系统         # Obsidian 配置、软件、工作流
05_生活健康         # 健身、饮食、旅行、习惯
06_认知思维         # 思维模型、决策框架、反直觉
07_资源模板         # 可复用的模板、表格、检查清单
08_归档             # 已完成 / 暂不沿用的内容
```

### 为什么是 9 个？

- 这是 PARA 方法 + 个人实际需求的折中产物
- 不像 PARA 那样只分 Projects/Areas/Resources/Archives，而是按用户实际关心的**生活领域**划分
- 编号前缀（00~08）的目的是强制排序：在 Obsidian 的文件浏览器中保持固定顺序，不因新建文件而打乱
- 每个顶层目录都包含一篇 `00_xxx导航.md`，作为该领域的入口页（也是给 Ai Agent 的上下文锚点）

### 为什么不采用标签 / Dataview 替代目录？

- 标签和 Dataview 适合**已建立结构后的检索**，不适合**初期模糊存放**
- 文件系统的目录对 AI Agent 更友好——智能体可以通过路径推断语义，不需要先理解 Dataview 查询语法
- 用户本人也依赖文件树导航

## 4. 交易日志设计：第三层结构

交易日志在整个知识库中的层级是：

```
03_财富投资
  └─ 02_交易执行            ← 第二层：主题目录
       ├─ 01_交易日志/       ← 第三层：单笔交易记录
       ├─ 02_周期复盘/       ← 第三层：周/月总结模板
       └─ 03_交易规则/       ← 第三层：纪律与规则
```

### 为什么是第三层？

- 第一层是「领域」（财富投资）
- 第二层是「主题」（交易执行）
- 第三层是「具体功能」（交易日志 / 周期复盘 / 交易规则）

用户明确指定的策略：**交易日志在第三层**。这是合理的，因为把单日交易记录直接放在顶层会污染导航，放在第二层又过于笼统。

### 交易日志内部的命名约定

- 单日日志：`2026-06-28 交易日志.md`（前导日期方便排序）
- 周总结：`2026-26周交易总结.md`
- 月总结：`2026-06月交易总结.md`
- 每个文件以 `#交易记录` 或 `#复盘` 标记开头，便于通过搜索聚合

### Dataview 索引

交易中心的 `00_交易中心.md` 包含一个 Dataview 查询，从 `03_财富投资/02_交易执行/01_交易日志/` 中抓取所有带 `交易日志` 标签的笔记。

---

## 5. 每个目录的导航约定

```text
每个目录下的第一篇笔记，约定命名为 00_xxx导航.md
```

例如：
- `00_Inbox/00_收件箱.md`
- `01_个人成长/00_个人成长导航.md`
- `03_财富投资/02_交易执行/00_交易中心.md`

这是一种**自解释**的设计：
- AI Agent 进入任意目录，第一眼就能找到说明文件
- Obsidian 的文件排序规则使其始终排在首位
- 导航笔记中包含该目录的用途、子目录列表、链接到其他领域的出口

### 导航笔记的推荐格式

```yaml
---
title: 领域名称
tags:
  - 导航
  - 索引
  - MOC
aliases:
  - 别名列表
---
```

内容至少包含：
1. 一句话说明该领域做什么
2. 子目录 / 子笔记的链接列表
3. 回到 [[首页]] 的链接
4. （可选）Dataview 查询汇总

| 可见性 | Public |

| 凭证方式 | Git Credential Manager（与浏览器登录联动） |

### .gitignore 策略

```gitignore
# 不跟踪本地界面状态
.obsidian/workspace.json
.obsidian/workspace-mobile.json

# 不跟踪缓存与回收站
.obsidian/cache/
.trash/

# 不跟踪临时文件
*.tmp
*.bak
~$*
```

这个策略的意图是：
- **保留** Obsidian 配置（主题、插件、设置），方便克隆后直接使用
- **不跟踪** 本地工作区状态（哪些标签页打开、窗口布局），避免无意义冲突
- 如果需要多人/多设备同步，这样配置最干净

## 7. 对 AI Agent 的使用建议

如果你（AI）要在这个知识库中执行任务，请遵循以下规约：

### 7.1 理解路径

- 根目录是知识库的入口，`首页.md` 是总导航
- 路径前缀反映了领域归属，例如 `03_财富投资/02_交易执行/01_交易日志/` 表示「财富投资 → 交易执行 → 日常记录」
- 不要轻易改变目录的编号前缀或结构，除非用户明确要求重构

### 7.2 写笔记的约定

- **交易日志**：按日期命名，放在 `03_财富投资/02_交易执行/01_交易日志/`
- **周/月总结**：放在 `03_财富投资/02_交易执行/02_周期复盘/`
- **交易规则**：放在 `03_财富投资/02_交易执行/03_交易规则/`
- **不确定放哪的内容**：先放到 `00_Inbox/`
- **模板类**：放在 `07_资源模板/`
- **项目笔记**：放在对应的领域目录下（02_专业事业 / 01_个人成长 等）

### 7.3 阅读优先

在创建新笔记之前，先查看：
1. `首页.md` — 了解全局结构
2. 目标目录下的 `00_xxx导航.md` — 了解该领域的边界和已有内容
3. 相关的模板文件 — 了解内容格式约定

### 7.4 编辑限制

- 不修改 `.gitignore`，除非用户要求变更同步策略
- 不修改 `.obsidian/` 下除 `workspace.json` 外的其他配置
- 不删除其他 AI Agent 创建的内容，除非用户明确要求或确认

## 8. 未来可能的扩展方向

- **生活管理主页**：一个聚合日常习惯、代办、健康数据的 Dashboard
- **学习主题索引**：按技能树组织的学习路径页
- **项目阶段复盘**：用于阶段性项目回顾的模板
- **周报自动化**：结合 Dataview 自动汇总当周交易与学习产出
- **Dashboard 首页**：用 DataviewJS 做可视化卡片布局

这些目前是「预留方向」，尚未真正实现，适合作为后续 AI Agent 的改进切入点。

## 10. 滚动引用功能（v1.1）

### 用途

打开任意笔记时，顶部自动显示一条与该笔记标签匹配的随机语录：
- 打开交易日志 → 显示交易纪律语录
- 打开学习笔记 → 显示学习原则
- 打开项目文档 → 显示工作规范
- 打开日记 → 显示人生信条

### 实现方式

由三个组件协作完成：

1. **引用数据库** `09_知识库管理/引用数据库.md`
   - 以 Obsidian 列表 + Dataview 内联字段 (`类别::` `语录::` `作者::`) 格式存储
   - DataviewJS 通过 `dv.pages().file.lists` 读取
   - 新增语录只需在文件中按格式添加一行

2. **DataviewJS 代码片段** `09_知识库管理/滚动引用模板片段.md`
   - 一段可复用的 JavaScript 脚本
   - 读取当前笔记的 `tags`，匹配「标签→类别」映射表
   - 在引用数据库中过滤出同类别的语录
   - 随机选取一条，以 `[!quote]` callout 格式渲染
   - 标签未匹配时，自动回退到按文件路径关键词匹配

3. **模板集成**
   - `交易日志模板`、`周总结模板`、`月总结模板` 顶部已内嵌该代码块
   - 其他笔记模板可按需从 `09_知识库管理/滚动引用模板片段.md` 中复制

### 使用前提

- 必须安装 **Dataview 社区插件**
- 代码块在编辑模式下不渲染，需在阅读/预览模式查看效果
- 每次打开/刷新笔记，会重新随机选取一条
