# 🧠 LEO 知识库构建思路

> 本文档是 LEO 知识库的完整架构说明，记录构建背景、设计决策、每个方向的结构规划以及使用约定。
> 如果你是一个 AI 智能体，请先阅读本文，再对库中的内容进行操作，并在操作后同步到github。

---

## 1. 构建原因

AI:
这个知识库的**出发点是一套人生管理系统**。
用户可以记录生活中的各个方面——成长、事业、财富、健康、认知、工具——并让它们在一个统一的结构中互相关联、长期沉淀。交易是第一个被深入设计的模块，但不是整个知识库的「主题」。
最初的契机确实是一个交易日志的需求：系统化记录每次交易，通过周/月总结复盘，沉淀交易纪律。但在整理过程中，发现原有的笔记结构无法承载更广泛的内容（学习笔记、项目记录、日常生活、素材图片散落各处），于是决定**从零规划**一个面向「生活·成长·事业·财富」一体化的个人知识库。

## 2. 核心设计原则

```
原则 1：先记录，再整理 —— 降低起步门槛
原则 2：沉淀可复用的信息，不保留临时状态
原则 3：模板尽量简洁，保证长期可坚持
原则 4：按人生方向分域，每个域按需决定内部层级
原则 5：能留空就先留空，不过度规划未开发的方向
```

## 3. 顶层架构：十大方向

整个知识库根目录下有 10 个编号顶层目录、一个 `首页.md` 入口，以及 `README.md`（GitHub 仓库说明）和 `pictrues/`（图片素材文件夹）。

### 3.1 方向总览

```text
00_Inbox           # 📥 收件箱 — 临时存放，待分类处理
01_个人成长         # 👤 个人成长 — 阅读、技能、自我提升
02_专业事业         # 💼 专业事业 — 工作、项目、履历、面试
03_财富投资         # 💰 财富投资 — 资产总览、交易日志、复盘规则
04_工具系统         # 🔧 工具系统 — 软件、工具、自动化、工作流
05_生活健康         # 🌿 生活健康 — 健身、饮食、旅行、习惯
06_认知思维         # 🧠 认知思维 — 思维模型、决策框架、复盘方法论
07_资源模板         # 📋 资源模板 — 可复用模板、清单、外部索引
08_归档             # 📦 归档 — 已完成 / 暂不沿用的内容
09_知识库管理       # ⚙️ 知识库管理 — 架构设计、数据库、代码片段
```

### 3.2 设计背景

- 编号前缀（00~09）强制 Obsidian 文件浏览器中的固定排序，不因新增文件打乱
- 借鉴 PARA 方法但按用户实际关心的**人生方向**划分，比纯粹按 Projects/Areas/Resources/Archives 更直观
- 每个顶层目录配一篇 `xxx导航.md` 作为入口索引（也是给 AI Agent 的上下文锚点）
- 标签和 Dataview 用于**已有结构后的检索和聚合**，目录树用于**初始导航和语义归属**

## 4. 分方向详述

### 4.1 00_Inbox · 收件箱 

**定位**：所有暂未分类的内容入口。收到新信息、想法、素材，先丢进这里，日后再整理到正式目录。

**当前结构**：
```
00_Inbox/
  ├─ 收件箱.md              # 入口说明
  ├─ 模板/                      # 可复用模板
  │    ├─ 收件箱条目模板(v1.0).md
  │    └─ 日常记录模板.md
  └─ 2026-6、7/                # 按时间段分组的临时内容
       ├─ C盘清理.md
       └─ 歌单整理.md
```

**命名约定**：临时内容可按年月分组（如 `2026-6、7/`），定期回顾并清空到正式目录。

**模板体系**：两种模板按事件重要程度区分——重要事件用 `收件箱条目模板(v1.0)`，普通事件用 `日常记录模板`。两者均带 `inbox` 标签，可通过 Dataview 统一检索。

**定位**：管理账户、资金分配、基金持仓、交易执行与复盘。

```
03_财富投资/

       ├─ 模板/                             # 可复用模板
       │    ├─ 交易日志模板(v1.0).md       # 带滚动引用的日交易模板
       │    ├─ 周总结模板(v1.0).md
       │    └─ 月总结模板(v1.0).md
       ├─ 01_交易日志/                     # 三级：单笔交易记录
       ├─ 02_周期复盘/                     # 三级：周/月复盘笔记
       └─ 03_交易规则/                     # 三级：纪律与资金管理
            └─ 交易规则清单.md
```

**交易日志内部的命名约定**：
- 单日日志：`2026-06-28 交易日志.md`（日期前缀，自然排序）
- 周总结：`2026-26周交易总结.md`
- 月总结：`2026-06月交易总结.md`
- 每个文件以标签 `#交易日志` 或 `#复盘` 标记，便于 Dataview 聚合

**Dataview 索引**：`交易中心.md` 包含 Dataview 查询，从 `01_交易日志/` 抓取所有交易日志笔记。

## 5. 已实现的技术功能

### 5.1 滚动引用轮播（v1.3）

打开任意笔记时，顶部自动滚动显示与该笔记标签匹配的随机语录，每 5 秒切换一条。

| 组件 | 文件 | 说明 |
|------|------|------|
| 数据源 | `引用数据库.md` | Obsidian 列表 + 内联字段格式，DataviewJS 通过正则解析 |
| 逻辑代码 | `滚动引用模板片段(v1.0).md` | `setInterval` 定时轮播，时间种子伪随机，闭包 DOM 引用 |
| 已部署 | 交易日志/周总结/月总结模板 | 模板顶部内嵌压缩版代码 |

技术要点：
- Dataview 不能解析列表续行中的内联字段，代码改用正则从 `item.text` 提取
- 使用 `Date.now()` 时间种子替代 `Math.random()` 避免缓存复用
- `setInterval(fn, 5000)` 每 5 秒通过闭包引用的 DOM 对象直接更新 `innerHTML`
- 模板文件中的代码为压缩版；可读版在 `滚动引用模板片段(v1.0).md` 中维护

### 5.2 模板文件版本命名

所有可复用的模板文件名附带版本号后缀 `(v1.0)`，方便迭代时区分。实际创建的笔记不使用版本后缀。

```gitignore

# 不跟踪缓存与回收站
.obsidian/cache/
```

## 7. 对 AI Agent 的使用规约

如果你（AI）要在这个知识库中执行任务，请遵循：

### 7.1 理解路径

- 根目录是知识库入口，`首页.md` 是总导航
- 路径前缀反映方向归属：`03_财富投资/02_交易执行/01_交易日志/` → 「财富投资 → 交易执行 → 日常记录」
- 不要改变目录编号前缀或顶层结构，除非用户明确要求重构

### 7.2 写笔记的约定

- **不确定放哪**：先放到 `00_Inbox/`
- **交易日志**：`03_财富投资/02_交易执行/01_交易日志/`，按日期命名
- **周/月总结**：`03_财富投资/02_交易执行/02_周期复盘/`
- **交易相关规则**：`03_财富投资/02_交易执行/03_交易规则/`
- **项目笔记**：对应领域目录下（`02_专业事业`、`01_个人成长` 等）

### 7.3 阅读优先

在创建新笔记之前，先查看：
1. `首页.md` — 全局结构
2. 目标目录下的导航文件 — 领域边界和已有内容
3. 相关模板文件 — 内容格式约定

### 7.4 编辑限制

- 不修改 `.gitignore`，除非用户要求变更同步策略
- 不修改 `.obsidian/` 下除 `workspace.json` 外的其他配置
- 不删除用户或之前 AI 创建的内容，除非用户明确要求
